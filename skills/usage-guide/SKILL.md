# 院校数据收集系统 — 使用指引

本系统用于自动收集高校设计类专业的结构化数据（专业信息、申请要求、学费、教授等）并翻译为多语言版本。

## 两种核心操作

| 操作 | 说明 | 是否重建 site_map |
|------|------|------------------|
| **爬取** | 三阶段全流程：站点发现 → 数据提取 → 聚合校验 | 是，覆盖重建 |
| **更新** | 增量更新：在已有 site_map 基础上刷新数据 | 否，保留现有 |

> **简单记忆**：爬取 = 从头来，更新 = 只刷新数据

## 命令示例

### 爬取（三阶段流程）

```
爬取 hbksaar
爬取 hbksaar、hfg-offenbach
重新爬取 kunsthochschule-berlin-weissensee
重新爬取 hbksaar、hfg-offenbach
重爬 fill-rate 最低的 5 所
重爬最久未探索的 3 所
全量更新
全量更新德国学校
```

### 更新（增量更新）

```
更新 hbksaar 数据
更新 hbksaar、hfg-offenbach 数据
更新 fill-rate 最低的 5 所
更新最旧的 3 所学校
更新全部
```

### 其他操作

```
提取这个页面: https://www.xxx.de/program
搜索德国设计院校
搜索英国工业设计专业的学校
```

## 条件筛选

多校操作可以用条件筛选代替指定学校名称：

- `fill-rate 最低的 N 所` — 数据质量最差的学校
- `最旧/最久未更新的 N 所` — 最长时间没更新的学校
- `专业最多的 N 所` — 按专业数量排序
- 可以加国家限定：`更新所有英国学校`、`重爬 fill-rate 最低的 3 所德国学校`

## 数据产出

每所学校完成后会生成：
- `site_map.md` — 网站结构地图（仅爬取时生成/更新）
- `_index_EN/ZH/DE.md` — 学校级数据（多语言）
- `programs/{专业}/_index_EN/ZH/DE.md` — 专业级数据（多语言）
- `university_profile_EN/ZH/DE.md` — 学校摘要文档
- `collection_status.yaml` — 全局状态更新

## 常见问题

**Q: 爬取和更新有什么区别？**
爬取会重新扫描整个网站、重建 site_map、重新提取所有数据，适合首次收录或数据严重过时的情况。更新在现有 site_map 基础上刷新数据，速度更快，适合日常维护。

**Q: 单所学校要多久？**
取决于专业数量和网站复杂度。一所 3-5 个专业的学校通常需要几分钟。

**Q: 数据存储在哪里？**
`data/universities/{国家代码}/{学校缩写}/` 目录下，所有数据自动 git commit 并 push。
